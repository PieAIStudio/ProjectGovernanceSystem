# comm-coach 沟通教练

> 分析用户与 AI 的对话，提供基于证据的沟通改进建议。

---

## 设计理念 Design Philosophy

### 为什么需要这个技能？

**核心问题**：用户与 AI 沟通时常遇到：

- AI 理解偏差 → 需要多轮澄清
- 输出不符合预期 → 浪费时间重来
- 不知道问题出在哪 → 无法改进

**根本原因**：

1. **模糊表达**：用户习惯人类交流的"懂我"默契，AI 无法脑补
2. **上下文缺失**：用户假设 AI 知道背景，实际每轮对话是独立的
3. **缺乏结构**：没有使用框架组织需求，导致遗漏关键信息

### 设计目标

| 目标         | 方法                       |
| ------------ | -------------------------- |
| **发现问题** | ToT 8 分支多维度扫描对话   |
| **验证可靠** | 强制搜索 + 交叉验证防幻觉  |
| **可操作**   | 提供模板和练习，不只是理论 |
| **可追溯**   | 输出链接，用户可随时查验   |

## 核心概念 Key Concepts

### ToT (Tree of Thoughts) 思维树

**是什么**：一种多角度探索问题的分析方法。像树的分支一样，从多个维度同时分析同一个问题。

**本技能如何使用**：

- 8 个分支同时分析用户消息
- 每个分支独立评分 (1-5)
- 最终筛选 Top 3-5 最重要的优化点

```
[用户消息]
    │
    ├── 模糊表达？
    ├── 上下文缺失？
    ├── 角色定义？
    ├── 约束声明？
    └── ... (共 8 个维度)
```

### SSDP (语义相似性动态剪枝)

**是什么**：合并语义相似的问题，避免重复输出。

**例如**：

- "模糊表达：用了'改一下'"
- "上下文缺失：没说改什么"
- → 合并为：「表达笼统，缺乏具体目标」

### 验证性搜索 vs 发现性搜索

| 类型       | 目的                           | 时机                   |
| ---------- | ------------------------------ | ---------------------- |
| **验证性** | 确认推荐的框架是否真实可靠     | Phase 2 方法论推荐时   |
| **发现性** | 主动发现用户可能不知道的新框架 | Phase 3 扩展知识网络时 |

### 交叉验证

**是什么**：用 2+ 个不同来源验证同一个框架，防止 AI 幻觉。

**为什么重要**：

- AI 可能编造不存在的框架名称
- 单一来源可能有偏差
- 用户可以点击链接自行核实

## 使用指南 SOP

### 何时使用？

- ✅ 觉得 AI 总是误解你的意思
- ✅ 想系统提升与 AI 沟通的效率
- ✅ 准备重要的 prompt 前想检查质量
- ✅ 学习沟通框架（如 CO-STAR、RACE）

### 如何触发？

在对话中说：

- `/comm-coach`
- `分析我的沟通`
- `怎么跟AI沟通更有效`
- `帮我检查这个 prompt`

### 输出示例

```markdown
## 对话分析 Conversation Analysis

### ToT 探索过程

- 角色定义 Role: 发现 2 处 → 影响分 4/5 ⭐
- 约束声明 Constraints: 发现 3 处 → 影响分 5/5 ⭐⭐
- ...

### 发现的优化点

#### 1. 约束声明缺失 (消息 #5)

**原文引用**: "帮我重构这段代码"
**问题**: 没说什么不能改
**改进**: "约束：不能改变函数签名，不能增加新依赖"

## 推荐方法论

### RACE Framework

**验证来源**:

- [链接1] GovTech Singapore 官方文档
- [链接2] Medium 企业实践案例
```

### 如何验证 AI 没有胡说？

每个推荐的框架都会附带**网页链接**，你可以：

1. 点击链接查看原始来源
2. 确认框架确实存在且定义一致
3. 如果链接失效或内容不符，说明 AI 可能有幻觉

## 框架速查 Framework Quick Reference

### AI 沟通框架

| 框架        | 用途       | 核心要素                                       |
| ----------- | ---------- | ---------------------------------------------- |
| **CO-STAR** | 全面提示   | Context-Objective-Style-Tone-Audience-Response |
| **RACE**    | 角色驱动   | Role-Action-Context-Expectation                |
| **CoT**     | 复杂推理   | "让我们一步步思考"                             |
| **ToT**     | 多角度探索 | 分支→评估→筛选                                 |

### 人际沟通框架

| 框架     | 用途        | 核心要素            |
| -------- | ----------- | ------------------- |
| **SCQA** | 结构化汇报  | 情境-复杂-问题-答案 |
| **PREP** | 说服论证    | 观点-理由-例子-观点 |
| **STAR** | 讲故事/面试 | 情境-任务-行动-结果 |
