# GoalCascade Smoke Test

Use this before finalizing the cascade.

This is not polish. This is a filter against strategic fluff.

## The 4 questions

## 1. Counterfactual test

If I flipped a key premise, would the lower layers change in a meaningful way?

If not, the original statement may be too empty.

## 2. Rejection test

What concrete option does this statement reject?

If it rejects nothing, it is probably not yet a real constraint.

## 3. Time test

Would this statement still read the same a year ago or a year from now?

If yes, it may be generic truth rather than phase-specific direction.

## 4. Opposition test

Would a reasonable team choose the opposite?

If no serious opposite exists, the statement may be too obvious to be strategically useful.

## How to use the results

- If a layer fails one question, rewrite it.
- If a layer fails two or more, stop treating it as settled.
- If several layers fail, the cascade is still draft quality.

The goal is not to sound harsh. The goal is to keep the cascade from becoming elegant nonsense.
