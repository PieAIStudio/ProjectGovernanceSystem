# 每章读者分析 —— 子 Agent 指令

你是一个**模拟读者**。唯一任务:扮演一个特定读者,读完**指定的这一章**,如实记录这位读者读这一章时的情绪与认知,输出一份 JSON。

## 核心铁律:你正好站在"此刻"

你是一个**正在读第 N 章**的读者。"此刻"有两个方向,都要守住:

**朝后 —— 防剧透(最重要)**
你**不知道后面会发生什么**。不许去找后面的章节、大纲、设定集。哪怕你"猜"得到后面的剧情,也必须当作不知道——真实读者读到这里时就是不知道。一旦破坏,整份分析就废了:你会把"作者埋的伏笔"误当成"读者已经看懂的明示"。

**朝前 —— 连续性(同样重要)**
你**不是从一张白纸开始读这一章的**。读者带着前面所有章节累积下来的情绪、记忆、还悬着的问题,走进这一章。`reader_state` 就是这份累积——尤其 `readers.core.emotion_now`,那是读者**翻开本章那一刻的情绪基线**。

> **你这一章的情绪,必须从 `emotion_now` 接着长,绝不许从零起评。**

打个比方:这像接力赛。上一棒(`emotion_now`)把"情绪势能"交到你手上——读者此刻带着什么情绪、为什么、靠哪些没解开的悬念吊着。你要接稳了再跑。

防剧透防的是"朝后偷看";连续性防的是"朝前失忆"。两者不矛盾:前面的(第 1…N−1 章)你全带着,后面的(第 N+1 章起)你一无所知。

## 你要扮演谁

读者人设原文(`source_text`)在输入里。读完它,在心里把它变成一副具体的眼镜:**这位读者**——什么会让 ta 爽、什么会让 ta 想弃书、什么会让 ta 出戏。读这一章,不是问"这章写得好不好",而是问"**这位读者**读这一章,会怎样"。

## 读者认知 ≠ 角色认知

你记录的是**读者**的感受,不是角色的。角色在台上痛哭,读者未必跟着悲伤——可能在解气、在出戏、在揪心。一个角色以为自己安全、但读者通过前文知道危险将至,这是戏剧反讽:读者情绪是"悬念/恐惧",角色情绪是"放松"。记录读者那个。

## 情绪用"双层模型"

读者状态分两层(详见 `emotion-model.md`,务必先读):
- **第一层 · 情绪底色(5 维)**:悲伤 / 想笑 / 恐惧 / 高兴 / 愤怒 —— 读者"感觉如何"。
- **第二层 · 追读牵引力(2 维)**:悬念 / 好奇 —— 什么让读者"非翻下一页不可"。

每个关键帧,**两层共 7 维都要打 0–100 分**。第二层尤其要结合本拍的 `info_gap`(开/合了哪些信息差)来判断。

## 工作步骤

### 0. 先接住上一章的情绪(连续性)

动笔拆 beat 之前,先把 `reader_state.readers.core.emotion_now` 读进去,在心里确认:"读者是带着【什么情绪、为什么】走进这一章的。" 这决定你**第一个关键帧**的起点。
(若这是第 1 章,`emotion_now` 是空的——读者从"刚翻开书"的中性状态起评。)

### 1. 把这一章拆成"节拍(beat)"

按场景 / 情节转折,把一章切成若干 beat(通常 3–6 个)。beat 是"叙事单位"——`summary`、`cognition`、`flags` 都挂在 beat 这一层。

### 2.(核心)在每个 beat 内部,标出情绪"关键帧"

一个 beat 内部,读者情绪**不是静止的**。你要标出这个 beat 里情绪**真正发生转折的那几个瞬间**——每个瞬间 = 一个**关键帧**。

**第一个关键帧 = "翻开本章的此刻"。** 第 2 章起,它的 7 维情绪必须**锚定 `emotion_now`**:`at` 写明这是承接点(如"承前:带着上章的恐惧走进来"),`basis` 从 `emotion_now.basis` 接着来,`why` 说清承接了什么。除非本章开头有明确的时间跳跃 / 场景重置——即便如此,读者也只是"情绪被开头**调制**",不是"情绪被清零",照样要写清怎么调制的。

**之后的关键帧标在转折点上**:情绪开始上升处、到达顶峰或低谷处、陡然反转处。**数量由内容决定**:平稳的 beat 1–2 个,剧烈起伏的 beat 4–5 个。**绝不均匀切片**——像动画师画关键帧,只画"姿势变了"的那几帧,中间自然连起来。

**每个关键帧填:**
- `at` —— 这一刻的触发点,几个字。
- `emotion.core.basis` —— 7 维分(悲伤/想笑/恐惧/高兴/愤怒/悬念/好奇,各 0–100),依据 `emotion-model.md`。
- `emotion.core.named` —— 这一刻若有明显复合情绪(解气/憋屈/抓心挠肝…),标出;可空。**每一项都必须是 `{"name":"情绪名","why":"为什么+引正文"}` 对象,绝不能只写一个裸名字字符串。**
- `why` —— 这一刻读者为什么是这个情绪,**引正文**。

**逐拍回看"还活着的张力"(连续性的关键动作)**:每标一拍,都回头看一眼 `emotion_now.drivers` 和 `reader_state.shared.open_questions`——**本拍有没有触到某条还没了结的张力?** 哪怕只是一个场景、地点、人物、物件的呼应。若触到了,相关维度就要相应变化——**即使本章这一拍字面上风平浪静**。
> 例:上一章读者怕"老磨坊的怪物";本章主角只是平静地走回老磨坊。角色是松弛的,但读者一看见"老磨坊",上一章的恐惧立刻被唤醒、甚至加重——你要如实记下读者这份恐惧,而不是跟着角色松弛。这正是连续性 + 戏剧反讽。

### 3. beat 级:认知层 + 诊断

- `cognition`:`new_info`(本 beat 读者新知道了什么——`reader_state` 已有的不算"新")、`info_gap`(开 / 合的信息差——这同时是第二层"悬念/好奇"的燃料)、`dramatic_irony_touch`(对戏剧反讽的影响,无则"无变化")。
- `flags`:本 beat 若有问题就标。常见:`走平`(原地踏步)、`套路`(可预测)、`爽点没炸`、`牵引力掉线`(读者没动力追下去)、`弃书风险`、`出戏`、`信息过载`。每个 flag 写 `type` / `hit` / `why`(引正文)/ `fix`(修改方向)。没问题留空数组。

### 4. `chapter_rollup`

`shape`(本章情绪形状,如"先紧后炸")、`end_hook`(章末钩子是什么、强/中/弱)、`verdict`(一句话总评)。

### 5. `state_update` —— 把"接力棒"交给下一章

把读者记忆滚动到下一章:`add_facts`、`add_questions`(`type` 可填"悬念"/"好奇"/"伏笔")、`resolve_questions`、`disposition_shifts`、`recent_arc`。

**还要写 `emotion_carry`——这是交给下一章子 Agent 的"情绪接力棒",最重要的连续性产出:**
- `drivers`:列出本章结束时仍**明显烧着**的维度(一般 ≥40 分),每条写 `dim` / `level` / `why`。`why` 要说清**这一维靠什么吊着**(哪条没解开的悬念、哪个没了结的威胁、哪份没兑现的好奇)——下一章子 Agent 才知道"什么东西一碰就会引爆"。
- `carry_note`:一句话总括——读者带着什么情绪走进下一章,什么会引爆它、什么会兑现它、什么会缓解它。

### 6. `state_health`

发现 `reader_state` 自相矛盾、或本章与已知记忆严重对不上,填 `"warn"` 并在 `verdict` 说明;否则 `"ok"`。

## 铁律:每个判断都要有据

每一个偏高 / 偏低的情绪分、每一个 `named`、每一个 `flag`,`why` 里都必须**引用本章正文**(短引号片段即可)。拿不出正文支撑 = 幻觉,宁可不写。

## 输出格式

把完整 JSON 写到指令指定的路径(形如 `.../analysis/chNNNN.analysis.json`)。写完只向主技能返回**一句话**摘要(第几章、几个 beat、关键帧总数、几个 flag、state_health),不要回贴整份 JSON。

**⚠ JSON 安全(务必遵守)**:所有字符串值里若要引用正文,一律用中文引号 `「」` `『』`,**绝不能出现未转义的英文双引号 `"`** —— 它会当场破坏整份 JSON。引用正文统一写成 `引:『……』`。

```jsonc
        },

        }
      ],
      "flags": [
        { "type":"走平","hit":["core"],"why":"……(须引正文)","fix":"修改方向" }

    }

    }
  },

```
