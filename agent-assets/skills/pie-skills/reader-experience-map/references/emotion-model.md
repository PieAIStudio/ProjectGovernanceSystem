# 情绪模型:双层 —— 情绪底色 + 追读牵引力

读者的内心状态用**两层**维度刻画。分析时,在每个**关键帧**(见 `subagent-instructions.md`)上为每一维打 **0–100** 分。

**为什么分两层** —— 读者读小说时,脑子里有两种不同的东西在动:
- 一种是**"感觉如何"**:悲伤、好笑、害怕…… 这是**情绪底色**。
- 一种是**"想不想翻下一页"**:被悬着、被勾着想知道…… 这是**追读牵引力**,它不是"感觉",是把读者往前拽的力。

两者混在一起会看不清(一本书可以"读着不爽却停不下来",也可以"读着舒服却没动力追")。所以拆成两层,分别打分、分别画线。

## 第一层 · 情绪底色(5 维)

读者**"感觉如何"**。这些是带正负色彩的felt emotion。

| 维度 | 含义 | 什么拉高 | 什么拉低 |
|---|---|---|---|
| 悲伤 | 难过、怅惘、心疼 | 喜爱的角色受苦、失去、遗憾 | 处境好转、被安慰 |
| 想笑 | 觉得好笑、被逗乐(≠ 高兴) | 妙语、荒诞、机智、反差、黑色幽默 | 笑点哑火、沉重且无趣 |
| 恐惧 | "怕"——面对具体可怕之物想逃 | 怪物现形、威胁逼到眼前、死物有了动静 | 威胁消失、回到安全 |
| 高兴 | 开心、欣慰、满足(好事发生) | 喜爱的角色得偿所愿、温暖时刻 | 挫败、坏消息 |
| 愤怒 | 义愤、来气、不平 | 反派作恶、不公、角色被欺负 | 恶人被清算、不公被纠正 |

## 第二层 · 追读牵引力(2 维)

什么让读者**非翻下一页不可**。这两维不是"感觉",是叙事把读者往前拽的"引擎"(叙事学称 narrative interest / epistemic emotion——认知型情绪)。

| 维度 | 含义 | 什么拉高 | 什么拉低 |
|---|---|---|---|
| 悬念 | "悬"——对**不确定的未来结果**坐立不安 | 危险逼近、倒计时、计划成败未卜、章末钩子 | 结果揭晓、尘埃落定 |
| 好奇 | "想挖"——发现一个**明确的信息缺口**,急着想知道被藏起来的真相 | 抛出谜团、点明"这里有内情"、给一点勾人的线索 | 真相揭示、缺口被填上 |

**0–100 量纲**:0 = 完全没有;30 = 隐约;60 = 明显;90 = 强烈。

## 三组最易混的维度,务必分清

- **悬念 vs 恐惧**(跨层)—— 悬念是"**悬**":对**不确定的结果**坐立不安(计划会不会成、买的车有没有坑)。恐惧是"**怕**":面对一个**具体的、可怕的东西**的本能反应,想逃(怪物从黑暗里现形)。两者常同时出现,但峰值位置不同:悬念冲突时悬念高、恐惧低;可怕之物逼到眼前时恐惧爆表。

- **悬念 vs 好奇**(第二层内部,最关键)—— 都让读者想往下读,但方向相反。**悬念朝前**:读者在**等一个还没发生的结果**(「丁一会不会被识破」「钱会不会打水漂」)。**好奇朝后/朝内**:读者发现**有个已经存在、却被藏起来的真相**,想挖出来(「罗斯柴尔德到底真不真」「秦若什么来头」「这局怎么设的」)。判分前自问:读者此刻是**在等一个未来结果**(悬念),还是**想知道一个已存在却被藏起的真相**(好奇)?

- **想笑 vs 高兴** —— 黑色幽默 = 想笑高、高兴低(在惨事里发笑)。两者独立,不要混。

## 第二层与"认知层"是咬合的

第二层不是凭空打分。技能在每个 beat 的 `cognition.info_gap` 里记录**开了哪些信息差、合了哪些**——那就是悬念/好奇的"燃料清单":
- **悬念** 由"开着的、指向**未来结果**的问题"喂养;
- **好奇** 由"开着的、指向**隐藏真相**的信息缺口"喂养。

给第二层打分前,先看本拍 `info_gap` 开/合了什么。一条重要规律(Loewenstein 信息差理论):**好奇的强度取决于缺口够不够"具体"**——模糊的"还有很多没交代"激不起好奇;明确到一个点的"那笔钱到底哪儿去了"才真正勾人。缺口越具体、读者越被"喂"过一点线索,好奇越强。

## 合成与渲染

- **第一层**有正负:正向(高兴 + 想笑)、负向(悲伤 + 恐惧 + 愤怒)。渲染时可算"情绪净值"。
- **第二层**不分正负,它是纯粹的"牵引强度"。渲染时算"追读牵引力"。
- 两层**分开画**:一条曲线管"读着爽不爽",一条管"抓不抓人"。

## 复合情绪 = 维度的组合

解气、憋屈这些**不是独立维度**,是若干维度的组合 / 轨迹。某个关键帧若出现明显的复合情绪,在该关键帧的 `named` 字段标出名字 + `why`。常见拆法:

| 复合情绪 | ≈ 维度组合 / 轨迹 |
|---|---|
| 解气 | 高兴↑ + 愤怒↓(积压的愤怒被清算)+ 悬念↓;**须接在一段低谷之后** |
| 憋屈 | 高兴低 + 悲伤微 + 愤怒被压住(想发作而不能) |
| 毛骨悚然 | 恐惧↑ + 一种冷而静的诡异感(死物有了生命迹象之类) |
| 爽 | 高兴↑ +(想笑或解气);期待被满足或超额满足 |
| 感动 | 高兴 与 悲伤 同时偏高(温暖的泪点) |
| 义愤 | 愤怒↑,且指向反派的不义 |
| 揪心 / 担忧 | 悬念↑ + 悲伤微 + 高兴低;针对喜爱角色的处境 |
| 抓心挠肝 | 好奇↑ + 悬念↑ 同时拉高 —— 最强的"放不下、必须追下去" |
| 尴尬 / 出戏 | 想笑↓ + 轻微悬念;角色行为让读者替其难堪 |

**轨迹型复合情绪**(尤其"解气"):判断时参考 `reader_state.recent_arc` 与本 beat 之前的关键帧——没有前面的低谷,就没有真正的解气。查表外的罕见情绪,可直接用 `named.name` 写自由文字。

## 铁律:每个判断都要有据

任何一维的高分、任何一个复合情绪,`why` 里必须**引用正文**作为依据(用中文引号 `『』`,见 `subagent-instructions.md` 的 JSON 安全规则)。拿不出正文支撑的判断 = 幻觉。
