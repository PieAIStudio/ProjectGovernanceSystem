# 模拟读者人设

本技能的工作是"扮演一个特定读者去读小说"。读者不是抽象的"大众"——必须是一个有**具体口味**的人,否则评判会退化成 AI 脑补的刻板印象。

每个读者人设 = 一段画像 + 一份**判断清单**(什么让 ta 爽 / 弃书 / 出戏)。

## P1:核心目标读者

P1 阶段只模拟这一个读者。

**来源**:每本小说的 `00_memory/idea_seed.md` 里都有"目标读者"画像。`prepare_run.py` 把这段原文抽取进 `run_plan.json` 的 `core_reader.source_text`;**子 Agent 读到后,自己在心里把它变成下面这份判断清单**,再据此读这一章——

- **likes(什么让 ta 爽)**:与 ta 的类型偏好、爽点期待相符的东西。
- **drops_when(什么让 ta 弃书)**:节奏、信息密度、主角行为方面 ta 不能忍的东西。
- **breaks_immersion(什么让 ta 出戏)**:破坏代入感的东西(假巧合、人物降智、说教)。

子 Agent 分析每一章时,**始终戴着这副人设的眼镜**:不是问"这章好不好",而是问"**这个读者**读这章会怎样"。

**示例 · deepfake-v1**
> 画像:25–40 岁都市读者;犯罪喜剧 / 黑色幽默爱好者;《疯狂的石头》《无名之辈》《唐探》受众;读网络连载。
> - likes:聪明的反转、解气的报应、小人物挣扎被理解、黑色幽默精准命中。
> - drops_when:节奏拖、笑点哑火、主角连续犯蠢且无成长。
> - breaks_immersion:巧合太假、人物降智、作者跳出来说教。

**示例 · steel-monster-v4**
> 由其自己的 `idea_seed.md` 实例化(现实悬疑 / 怪物追查题材 + 韩寒余华式弦笔的受众)。技能通用,人设逐书生成,不写死。

## 后续阶段(P2)

P2 将加入另外两个读者,数据结构已为多读者预留(`reader_state.readers` 是字典):

- **挑剔老书虫** —— 测套路 / 逻辑硬伤;复用框架 `/双审` 已有的"极严苛老书虫"人设。
- **路人 / 新读者** —— 测门槛 / 弃书点;耐心低,1–3 章决定去留。

P1 不实现这两个。
