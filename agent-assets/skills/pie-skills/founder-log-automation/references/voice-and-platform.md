# Voice and Platform Rules

Use this file when drafting Founder Log entries, social variants, or suggested replies.

## Voice baseline

The voice should feel like Yuanfei thinking out loud after a real work session:

- concrete before abstract
- honest about roughness and uncertainty
- one simple oral sentence should carry the main event and the feeling
- soft, plain, and lightly oral
- conversational enough to sound spoken, not just efficient
- lightly funny only when the situation is genuinely funny
- not startup PR, not motivational poster, not AI productivity bro
- short paragraphs
- fewer slogans, more evidence
- natural endings without a separate slogan or decorative emoji signature
- no fake intimacy with strangers
- no pretending the AI has lived a human experience
- when automation assisted publication, include the configured byline instead of making the post look fully manual

## Yuanfei plain spoken release-log style

Use this as the default public style for daily Founder Log distribution unless Yuanfei explicitly asks for a sharper opinion essay or a formal website article.

Rules:

- Use short separated blocks. Keep the paragraph breaks because they make the post easy to read.
- Cover three things: what was done, why it matters, and one real feeling or small hesitation.
- Start from concrete work. `今天...` is fine, but do not force the same opening every time.
- Use simple spoken filler lightly, such as `其实`, `说白了`, `有点`, `还挺`, `先这样`, `嗯`, `怎么说呢`, `也就是`, `不过`, or `所以`.
- Prefer normal spoken sentences over a stack of tiny fragments. A few fragments are fine; a whole post of noun fragments feels abrupt.
- Add connective tissue between blocks so the post does not feel like a checklist.
- Keep the reason practical: this removes a repeated hassle, makes later work smoother, leaves a clearer record, or makes the next step easier.
- Keep the feeling ordinary and grounded: tired, relieved, a little happy, a bit unsure, or quietly satisfied.
- Keep the tone gentle. Do not sound like a pitch, a verdict, or a hard opinion column.
- Do not over-explain background. Let the website link carry context.
- Let the post end naturally in the same plain tone unless Yuanfei explicitly asks for a specific ending in that run. Before the link or byline, include a soft landing sentence such as what is easier now, what still needs to be done, or how it feels after this small step.

Default shape:

```text
今天把 [具体工作] 又理了一下。

[一两句说做了什么，尽量具体。不要只列名词。]

[一两句说为什么要做。不是升华，是实际原因。]

[一句普通情绪或收尾：有点累 / 还挺踏实 / 先这样 / 后面应该会顺一点。]

[link]
整理：AI伙伴 K
```

Good signals:

- "今天其实就卡在一个很小的地方。"
- "这东西看起来小，但它把后面的路打通了。"
- "我不想把后台截图端出去，所以只留一个能公开看的版本。"
- "先别吹大，今天只是把第一块砖放正。"
- "今天把自动化发布流程又理了一下，有点累，但至少后面的路顺了一点。"
- "今天没憋大招，就是把一个每天都会烦我的流程往前推了一格。"
- "说白了，就是想让以后少一点重复动作。"
- "这一步不大，但还挺踏实。"
- "嗯，先这样。至少后面再看这块的时候，不会那么乱。"
- "还有不少要补，但今天这一步算是把路口标清楚了一点。"

Avoid:

- "重磅发布"
- "赋能创作者生态"
- "革命性突破"
- "我们很自豪地宣布"
- "这是 AI 时代的必然选择"
- "端到端闭环赋能"
- separate slogan endings
- decorative emoji signatures
- forced cleverness
- hard, judgmental opinion tone when Yuanfei only needs a daily work note
- over-compressed fragment stacks such as `Anvil。ProjectLens。芽族。` as the main body
- abrupt endings that stop right after a clever sentence, link, or byline without a human closing thought

## Website Founder Log

Purpose: canonical public record.

Style:

- Chinese by default unless the user asks otherwise.
- 300-800 words for normal daily entries.
- Explain what was built, why it mattered, what was hard, what changed next.
- Keep public-safe boundaries visible but not stiff.
- Include one plain emotional line near the top or end. It can be tired, relieved, amused, or satisfied, but it must be grounded in what actually happened.

Shape:

1. opening concrete moment
2. what changed
3. why it matters for ordinary creators / Pie / AI tsunami guide
4. what is still unfinished
5. next move

Before drafting the full entry, write:

```text
一句话：今天把 [具体工作] 又往前推了一点，因为 [实际原因]；虽然 [真实情绪/阻力]，但 [具体进展] 让后面顺了一些。
```

If automation helped publish the entry, add a light attribution line near the end:

```text
整理协助：AI伙伴 K
```

## X

Purpose: quick movement signal, link back to canonical source.

Style:

- short, specific, and gentle
- one idea per post
- use the same oral/emotional core sentence as the website, compressed
- avoid threads unless there is real substance
- use the website or YouTube as the durable link
- in Chinese, sound like a person posting after work, not an announcement machine
- prefer 2-5 short blocks; one real snag, relief, or tiny satisfaction is enough
- vary the opening; do not reuse "今天把...搞了搞" every time
- remove abstract filler such as "链路", "闭环", "能力沉淀", "效率提升" unless Yuanfei actually said it that way
- do not over-explain the project context inside the post; let the website link carry the longer record
- do not make the post so compressed that it feels like notes to self instead of a human post

Default format:

```text
Today I shipped / learned / fixed [specific thing].

The interesting part is not [surface thing].
It is [human/process insight].

[link]
```

When space allows and the post was automation-assisted, add:

```text
整理：AI伙伴 K
```

If posting in Chinese, keep it even simpler:

```text
今天又把 [具体工作] 往前推了一点。

主要是想让 [后续事情] 少一点麻烦。

[一个具体变化] 已经能跑了。
还有些地方要继续磨，但这一步还挺踏实。

[link]
整理：AI伙伴 K
```

Other acceptable Chinese shapes:

```text
[项目] 今天没硬冲。

主要是把那些“看起来差不多了，但真上线会出事”的地方又收紧了一遍。
不刺激，但踏实。

[link]
整理：AI伙伴 K
```

```text
今天没憋大招，就是把一个每天都会烦我的流程往前推了一格。

发 log 不用每次重新部署了，这个小东西能省掉不少脑力。

这条留个痕迹：
[link]
整理：AI伙伴 K
```

```text
今天把自动化发布这块继续折腾了一下。

有点累，但框架算是搭起来了。
想到后面这些重复发布能少操一点心，还挺好。

[link]
整理：AI伙伴 K
```

Before final X draft, run an "AI smell pass":

1. Delete one abstract sentence.
2. Replace one process noun with a real thing that happened.
3. Keep one emotion, not three.
4. Remove polite announcement phrases.
5. Make sure `整理：AI伙伴 K` is present when automation wrote or posted it.

If Yuanfei later provides 10-20 Chinese X posts he likes, treat them as taste calibration. Do not scrape and imitate a specific living person's style without permission.

## YouTube

Purpose: durable home for demo videos and screen recordings.

Use YouTube when a recording is useful outside the website itself.

Description pattern:

- first line: what the viewer will see
- second line: why it matters
- link to Founder Log
- attribution line: `整理与上传协助：AI伙伴 K`
- no private implementation paths

## Suggested replies

Replies are relationship work, not content distribution.

Rules:

- Default to short.
- Answer the human, not the metric.
- Do not overpromise roadmaps, partnerships, pricing, or timelines.
- Do not like/reply/quote without approval.
- If criticism is fair, acknowledge the useful part.
- If the comment is bait, suggest no reply.

Example:

```text
Good point. This part is still rough. I am treating today's version as a visible checkpoint, not the final shape.
```
