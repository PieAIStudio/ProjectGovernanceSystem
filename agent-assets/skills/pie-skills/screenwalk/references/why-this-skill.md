# 为什么这个技能存在

讨论"这个按钮放哪儿""这句文案怎么写"最容易失焦，因为**没有对齐到同一个用户动线**。每个参与者脑子里跑的"用户"可能都不一样。本技能强制把对话**锚定到真实屏**上——用户是谁、他在哪一屏上试图做什么、下一步去哪——让改进建议落在具体动线节点，而不是飘在"体验不好"的空中。

**它是什么**：截图/原型/真实页面驱动的动线评审工具。
**它不是什么**：代替真实用户研究；也不是写一份大 PRD。

## 与姐妹技能 `$goalcascade` 的分工

- screenwalk = 下游界面/路径问题（"这个具体屏对不对"）
- goalcascade = 上游方向问题（"我们到底在干嘛"）
- 如果本技能跑着跑着发现局部改动持续互相打架——那说明**问题不在屏上，在方向上**。停下局部讨论，推回 `$goalcascade`。

本技能只处理**已存在的屏和流**，不适合从零决定产品方向。
