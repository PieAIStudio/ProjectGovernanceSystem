# ScreenWalk 7 Questions

Use these exact questions on every screen.

## 1. 用户现在以为自己在哪里？

- What does the current screen seem to be, from the user's point of view?
- Look at title, layout, visible actions, and immediate context clues.

## 2. 用户此刻最想完成什么？

- What is the most likely task the user is trying to complete right now?
- Answer from the current step in the journey, not from abstract product goals.

## 3. 我希望用户此刻有什么感受？

- What feeling is the creator likely trying to produce here?
- Examples: curiosity, safety, urgency, trust, relief, excitement.

## 4. 用户第一眼最可能看到什么？

- Which visual element, CTA, text block, or empty state dominates first attention?
- This is about actual first-glance salience, not what the creator intended.

## 5. 用户最可能误解、犹豫或卡住在哪里？

- Where is the most likely confusion, hesitation, or wrong turn?
- If no serious friction appears, say why the path still reads as clear.

## 6. 为了达到目标，要改什么？

Do not answer with vague phrases like "optimize this."

Split changes by layer when relevant:

- **UI**: hierarchy, buttons, layout, density, contrast
- **UX**: next step, feedback, reversibility, flow clarity
- **Copy**: wording, comprehension, tone, emotional framing
- **Rules**: goals, victory, turns, resources, constraints
- **Game feel**: timing, impact, juice, readability, reward, state-change feedback
- **AI behavior**: role consistency, scene stability, prompt / harness reliability
- **Backend**: account state, room status, quotas, logs, risk controls
- **Monetization**: value exchange, quota pressure, upgrade timing, fairness
- **Accessibility**: readability, contrast, motion sensitivity, touch/click targets

## 7. 这个问题优先级是什么？

Use:

- `Blocker` - users cannot proceed, understand the core path, or finish the key step
- `High` - users can continue, but trust / comprehension / conversion / delight is meaningfully harmed
- `Later` - useful, but not essential to the first workable version

## Evidence note

When useful, tag whether a judgment comes from:

- `screenshot-visible`
- `flow-inference`
- `assumption`
- `real-user-data`

Do not inflate AI agreement into stronger evidence than it really is.
