# Clean Image Generation Rules

适用场景：当你要调用图像模型生成、编辑、抠图、批量出角色/场景/UI 素材时，先读这份规则。

核心原则：主线程负责策划和验收，生图线程保持干净。不要把长聊天、失败尝试、无关讨论都带进图像模型。

注意：没有可靠证据证明“长会话必然让图像起噪”，但长上下文、无关细节、失败轨迹会让模型更难抓住真正约束。对生产素材，按上下文污染风险处理。

## 0. 先判断宿主软件能怎么生图

不要假装自己有不存在的生图能力。先看你运行在哪个宿主里，再选路线：

| 宿主 | 首选路线 | 备用路线 |
| --- | --- | --- |
| Codex App，且当前会话有图像生成工具 | 用干净子代理/新短上下文直接调用图像模型；主线程只做规格、验收和入库 | 用 Codex Chrome Extension 打开网页生图 |
| Codex App，但没有图像生成工具 | 用 Codex Chrome Extension 打开网页生图 | 先整理最小任务包，等有生图能力的会话执行 |
| Copilot、Claude Code、其他 IDE/CLI AI | 用 Playwright Chrome Extension 打开网页生图 | 从终端启动一个干净 Codex CLI/App 会话，让 Codex 负责生图，再回收结果 |

网页生图优先级：

1. 优先用 ChatGPT GPTs gen：`https://chatgpt.com/g/g-6a1b30bb150c8191b4da05593b859068-gpts-gen`
2. 如果 ChatGPT 不可用、排队太久、或任务更适合 Gemini，再用 Gemini Gen：`https://gemini.google.com/gem/b404fa9ffb9a`

说明：截至 2026-05-31，OpenAI 官方文档确认 ChatGPT / GPT Image 支持生成、编辑、透明背景、参考图和较强指令跟随；本规则把 ChatGPT 放首位，是项目当前偏好和可用性策略，不是要求永久绑定某个供应商。

## 1. 先拆任务

- 先列出图片清单：每张图的用途、尺寸、主体、风格、不变量、交付路径。
- 把“同一角色/同一风格体系”的图归为一组，把不同角色、场景、UI、道具分开。
- 每张图只保留必要上下文；不要让一个图的失败细节污染下一张图。
- 如果任务已经在一个很长的会话里，先整理成新的短规格，再生图。

## 2. 用干净子代理

- 批量生图时，优先派子代理处理独立图片或图片组。
- 每个子代理只拿三样东西：统一风格胶囊、自己的图片规格、必要参考图；不要给完整主线程。
- 子代理完成后只回传：最终图、最终 prompt、关键参数、是否通过自检。
- 不要把子代理的探索过程、失败 prompt、长解释带回主线程；失败经验只总结成一句可执行调整。
- 如果没有子代理能力，就手动模拟：新开干净会话、干净浏览器对话或干净 Codex CLI/App 会话，只粘贴最小任务包。
- 如果通过网页生成，主线程不要在同一网页对话里连续塞很多不同资产；每 1 到 3 张图整理一次短任务包，必要时新开对话。

## 3. Prompt 要短、准、可复用

- 1 到 3 句讲清楚：主体、动作/构图、材质/光线、风格、尺寸用途。
- 明确锁定不变量：角色脸型、服装、姿势、画幅、背景、UI 留白等。
- 需要改图时写清楚：“只改 X，其他保持不变”。
- 采用固定顺序：用途/画布 -> 主体 -> 关键细节 -> 不变量 -> 避免项。
- 少用互相打架的风格词；负面词只写真正要避免的 3 到 6 项。

## 4. 保持一致性

- 有风格基准时，用固定的“风格胶囊”，不要每次重写一大段。
- 有角色/产品/材质一致性要求时，优先传参考图，而不是只靠文字描述。
- 不要迷信 seed；seed 可用于测试变量，但不能代替参考图和明确规格。
- 同一资产系列必须记录最终 prompt、参考图、模型、尺寸、参数、后处理步骤。

## 5. 小批量迭代

- 每个子代理一次处理 1 到 3 张图；多了就继续拆。
- 先生成小样，确认方向，再做高质量版本。
- 每次只改一个主要变量：构图、光线、表情、背景或材质。
- 如果出现噪点、脏边、风格漂移、奇怪文字，停止堆 prompt；改为新干净上下文 + 更短规格 + 必要参考图。

## 6. 验收再入库

- 看图，不要只相信 prompt。
- 检查：是否符合用途、风格是否干净、主体是否稳定、边缘是否可用、文字是否正确、背景是否满足需求。
- 需要抠图/透明背景/压缩/裁切时，把后处理作为独立步骤，不要混在同一个长生图回合里。
- 保存最终图和元信息，方便复现和之后扩展同一套资产。

## 7. 最小网页操作流程

当宿主不能直接调用图像模型时：

1. 用 Playwright/Codex Chrome Extension 打开 ChatGPT GPTs gen；不可用再打开 Gemini Gen。
2. 新建或清空当前生图对话，只粘贴“快速模板”里的最小任务包。
3. 一次生成 1 到 3 张同组图片；下载原图，不用截图代替资产。
4. 记录最终 prompt、网页工具、生成时间、下载路径、后处理步骤。
5. 把图放进项目资产目录后，再用项目自己的验证脚本和浏览器截图检查是否可用。

## 快速模板

给子代理的最小任务包：

```text
你是干净的图像生成子代理。不要继承主线程里的其他视觉设定。
风格胶囊：[固定风格，短句]
任务：[这张图的用途、主体、尺寸、背景、交付路径]
不变量：[必须保持的角色/构图/材质/文字]
避免：[只列真正不想看到的 3-6 项]
输出：最终图片、最终 prompt、模型/尺寸/参数、简短自检。
```

## 参考依据

- OpenAI image generation / ChatGPT Images docs: 支持生成、编辑、参考图、透明背景；复杂图可能需要较长等待。
- OpenAI Codex subagents: 把 noisy work 移出主线程，只返回摘要，避免 context pollution / context rot。
- Google Gemini Apps docs: 可生成和编辑图片；适合作为网页备用路线。
- Stability / Midjourney docs: 用清楚规格、少量负面词、参考图/风格引用来保持一致性；seed 只适合测试变量。
- 长上下文研究：无关信息和 distractors 会降低模型可靠性，所以生产生图应主动压缩上下文。
